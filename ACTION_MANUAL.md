# 📋 دليل الإجراءات اليدوية (Action Manual)

**كيفية النشر اليدوي على GitHub Actions متى أردت إرسال منشور**

---

## 🎯 نظرة عامة

يتيح لك هذا الدليل إرسال منشور يدوياً على حساب X (تويتر سابقاً) عبر GitHub Actions. هذا مفيد عندما:
- ⏰ تريد نشر تغريدة في وقت غير الوقت المبرمج تلقائياً
- 🔄 فاتك يوم وتريد تعويضه
- 🧪 تريد اختبار البوت دون انتظار الوقت المبرمج
- 🚀 تريد نشر تغريدة فوراً

---

## ⚡ البداية السريعة

### الخطوة 1: انتقل إلى GitHub Repository
1. افتح متصفحك واذهب إلى: `https://github.com/your-username/ramadan_sports_bot`
   (استبدل `your-username` باسم مستخدمك)

2. انقر على علامة تبويب **Actions** في القائمة العلوية

### الخطوة 2: ابحث عن Workflow
سترى قائمة بـ workflows. ابحث عن:
```
🌙 Auto Post Ramadan Hadith
```

### الخطوة 3: تشغيل يدوي
1. انقر على **Auto Post Ramadan Hadith**
2. انظر إلى الجانب الأيمن للصفحة
3. انقر على **Run workflow** ▼
4. تأكد من أن الفرع (Branch) هو **main**
5. انقر على **Run workflow** الزرقاء

---

## 📝 شرح مفصل

### ما يحدث عند التشغيل اليدوي:
1. **يبدأ تشغيل GitHub Actions** تلقائياً
2. **يتم تحميل الكود** من الفرع main
3. **تثبيت Python والمكتبات** المطلوبة
4. **تنفيذ البوت** بنفس طريقة التشغيل التلقائي
5. **نشر التغريدة** عبر X API
6. **تحديث سجل النشر** (`posted_log.json`)
7. **Commit التغييرات** إلى Repository تلقائياً

### الزمن المستغرق:
- ⏱️ **2-3 دقائق** تقريباً من بدء Workflow حتى النشر
- 📈 يعتمد على سرعة خوادم GitHub

## 🔧 المتطلبات الأساسية

قبل التشغيل اليدوي، تأكد من:

### 1. ✅ المفاتيح موجودة في Secrets
يجب أن تكون هذه المفاتيح مُخزّنة في **Repository Secrets**:
- `BEARER_TOKEN`
- `API_KEY`
- `API_SECRET`
- `ACCESS_TOKEN`
- `ACCESS_TOKEN_SECRET`

**للتحقق:**
1. انتقل إلى Repository → Settings
2. انقر على **Secrets and variables** → **Actions**
3. تأكد من وجود المفاتيح الخمسة

### 2. ✅ محتوى محدّث
تأكد أن `content.json` يحتوي على:
- ✅ تاريخ رمضان الصحيح (`ramadan_start`)
- ✅ المنطقة الزمنية المناسبة (`timezone`)
- ✅ وقت النشر (`post_time`)
- ✅ جميع الأحاديث والنصائح (30 يوم)

### 3. ✅ تاريخ اليوم ضمن رمضان
- البوت يتحقق من تاريخ اليوم الحالي
- إذا كان التاريخ خارج رمضان، لن يتم النشر
- يمكن تعديل `ramadan_start` في `content.json` إذا أردت اختبار خارج رمضان

## 🎬 دليل مصوّر

### الطريقة 1: من صفحة Actions الرئيسية
```
GitHub → Actions → Auto Post Ramadan Hadith → Run workflow → Run workflow
```

### الطريقة 2: من صفحة Workflow المحددة
```
GitHub → Actions → انقر على Auto Post Ramadan Hadith
→ انظر إلى اليمين → Run workflow ▼ → Run workflow
```

### الطريقة 3: باستخدام API (للمتقدمين)
```bash
curl -X POST \
  -H "Authorization: token YOUR_GITHUB_TOKEN" \
  -H "Accept: application/vnd.github.v3+json" \
  https://api.github.com/repos/YOUR_USERNAME/ramadan_sports_bot/actions/workflows/auto_post.yml/dispatches \
  -d '{"ref":"main"}'
```

## 🚨 استكشاف الأخطاء وإصلاحها

### ❌ الخطأ: "Workflow run failed"
**الأسباب المحتملة:**
1. **مفاتيح API خاطئة:** تحقق من Repository Secrets
2. **خطأ في Python:** قد تكون هناك مشكلة في الكود
3. **مشكلة في GitHub Actions:** تحقق من logs للتفاصيل

**الحل:**
1. اذهب إلى failed workflow → انقر على الخطوة الفاشلة
2. اقرأ error message
3. راجع الخطوات في `auto_post.yml`

### ❌ الخطأ: "Twitter API returned 401/403"
**السبب:** مفاتيح API غير صالحة أو منتهية الصلاحية

**الحل:**
1. أعد إنشاء مفاتيح X API من Developer Portal
2. عدّل Repository Secrets بالمفاتيح الجديدة
3. أعد تشغيل Workflow

### ❌ الخطأ: "Day already posted"
**السبب:** اليوم موجود بالفعل في `posted_log.json`

**الحل:**
1. افتح `posted_log.json`
2. احذف اليوم من القائمة
3. Commit التغييرات
4. أعد تشغيل Workflow

### ❌ الخطأ: "Ramadan hasn't started yet"
**السبب:** التاريخ الحالي قبل تاريخ `ramadan_start`

**الحل:**
1. عدّل `ramadan_start` في `content.json` لتاريخ سابق
2. Commit التغييرات
3. أعد تشغيل Workflow

## 💡 نصائح متقدمة

### 1. تشغيل يوم محدد
إذا أردت نشر يوم معين (غير اليوم الحالي):
1. افتح `content.json`
2. عدّل `ramadan_start` بحيث يكون اليوم المطلوب ضمن رمضان
3. Commit التغييرات
4. شغّل Workflow يدوياً

### 2. إعادة نشر يوم
إذا فشل نشر يوم وأردت إعادته:
1. افتح `posted_log.json`
2. احذف اليوم من القائمة
3. Commit التغييرات
4. شغّل Workflow يدوياً

### 3. جدولة نشر يدوي متكرر
يمكنك استخدام GitHub CLI لجدولة نشر يدوي:
```bash
# كل يوم الساعة 15:00
0 15 * * * gh workflow run "Auto Post Ramadan Hadith"
```

### 4. دمج مع أدوات أخرى
يمكن دمج GitHub Actions مع:
- **Slack:** إشعار عند النشر
- **Telegram:** إشعار عند النشر
- **Discord:** إشعار عند النشر
- **Email:** تقرير يومي

## ❓ الأسئلة المتكررة

### س: كم مرة يمكنني التشغيل اليدوي؟
**ج:** غير محدود، ولكن تحقق من حدود GitHub Actions (2000 دقيقة/شهر للمجاني)

### س: هل أحتاج إلى تحديث الكود قبل التشغيل؟
**ج:** لا، Workflow يحمل أحدث نسخة من main تلقائياً

### س: ماذا لو أردت إيقاف Workflow أثناء التنفيذ؟
**ج:** انتقل إلى Workflow run → انقر على **Cancel workflow**

### س: هل يمكنني تشغيل يوم غير اليوم الحالي؟
**ج:** نعم، عن طريق تعديل `ramadan_start` في `content.json`

### س: كيف أعرف أن النشر نجح؟
**ج:** تحقق من:
1. ✅ Workflow نجح (علامة خضراء)
2. ✅ ظهور التغريدة على X
3. ✅ تحديث `posted_log.json`

### س: ماذا لو كان رمضان قد انتهى؟
**ج:** عدّل `ramadan_start` لتاريخ قادم أو عدّل البوت
