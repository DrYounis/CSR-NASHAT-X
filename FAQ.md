## 🌟 أسئلة عامة

### س1: ما هو هذا المشروع؟
**ج:** بوت ذكي ينشر تلقائياً على X (تويتر سابقاً) محتوى يومي في رمضان يتضمن:
- حديث نبوي شريف من الأربعين الرياضية
- نصيحة رياضية مخصصة لرمضان
- 30 يوماً من المحتوى القيّم

### س2: هل المشروع مجاني؟
**ج:** نعم! المشروع مفتوح المصدر تماماً ومجاني للاستخدام الشخصي.

### س3: هل أحتاج معرفة برمجية؟
**ج:** معرفة أساسية بـ Python مفيدة، لكن الدليل مفصّل للمبتدئين. اتبع التعليمات خطوة بخطوة.

### س4: كم يستغرق الإعداد؟
**ج:** حوالي 5-10 دقائق للإعداد الكامل (إذا كانت لديك مفاتيح API جاهزة).

## 💻 التثبيت والإعداد

### س5: ما هي المتطلبات؟
**ج:** 
- Python 3.8 أو أحدث
- حساب X (Twitter) نشط
- حساب مطوّر X معتمد
- 19 KB مساحة تخزين

### س6: كيف أثبّت Python؟
**ج:**
- **Windows:** حمّل من [python.org](https://www.python.org/downloads/)
- **macOS:** `brew install python3`
- **Linux:** `sudo apt install python3 python3-pip`

### س7: هل يعمل على جميع أنظمة التشغيل؟
**ج:** نعم! البوت يعمل على:
- ✅ Windows 10/11
- ✅ macOS (Intel & M1/M2)
- ✅ Linux (Ubuntu, Debian, CentOS, etc.)
- ✅ Raspberry Pi

### س8: هل أحتاج سيرفر؟
**ج:** لا، يمكن تشغيله على حاسوبك الشخصي. لكن للتشغيل الدائم:
- استخدم VPS رخيص ($5/شهر)
- أو Raspberry Pi في المنزل
- أو خدمات سحابية مجانية

## 🔑 مفاتيح X API

### س9: كيف أحصل على مفاتيح API؟
**ج:** راجع ملف `API_KEYS_GUIDE.md` للشرح المفصّل. باختصار:
1. سجّل حساب مطوّر على developer.twitter.com
2. أنشئ Project + App
3. فعّل صلاحيات "Read and Write"
4. احصل على 5 مفاتيح

### س10: كم تكلفة مفاتيح API؟
**ج:** 
- **المستوى المجاني:** يكفي لهذا المشروع (500 تغريدة/شهر)
- **الأساسي:** $100/شهر (للاستخدام المكثف)
- البوت يحتاج فقط 30 تغريدة/شهر ✅

### س11: المفاتيح لا تعمل!
**ج:** تحقق من:
1. ✅ نسخ جميع المفاتيح الخمسة كاملة
2. ✅ لا توجد مسافات زائدة
3. ✅ صلاحيات "Read and Write" مفعّلة
4. ✅ Access Token تم إنشاؤه **بعد** تفعيل الصلاحيات

### س12: "403 Forbidden" - ماذا أفعل؟
**ج:** المشكلة في الصلاحيات:
1. اذهب إلى Developer Portal
2. Settings → App permissions
3. غيّر إلى "Read and Write"
4. **احذف** Access Token القديم
5. **أنشئ** Access Token جديد

## 🚀 التشغيل والاستخدام

### س13: كيف أبدأ التشغيل؟
**ج:**
```bash
python ramadan_bot.py
# اختر الخيار 1 للتجربة أولاً
# ثم الخيار 4 للنشر التلقائي
```

### س14: ماذا يحدث عند اختيار الخيار 4؟
**ج:** البوت سيعمل بشكل دائم:
- ينتظر الوقت المحدد (مثلاً 21:00)
- ينشر تغريدة اليوم تلقائياً
- يحفظ السجل لمنع التكرار
- يستمر حتى نهاية رمضان أو حتى توقفه

### س15: كيف أوقف البوت؟
**ج:** اضغط `Ctrl+C` في الـ Terminal

### س16: نسيت النشر ليوم واحد، ماذا أفعل؟
**ج:** استخدم الخيار 3:
```
👉 اختر رقم الخيار: 3
أدخل رقم اليوم (1-30): 15
هل أنت متأكد؟ (y/n): y
```

### س17: هل يمكن النشر أكثر من مرة يومياً؟
**ج:** نعم! طريقتان:
1. عدّل السكريبت لإضافة أوقات متعددة
2. شغّل نسختين بملفات content مختلفة

### س18: كيف أعرف أن البوت يعمل؟
**ج:** 
- شاهد الرسائل في Terminal
- تحقق من حسابك على X
- راجع ملف `posted_log.json`

## 🐛 المشاكل التقنية

### س25: "ModuleNotFoundError: No module named 'tweepy'"
**ج:** المكتبات غير مثبتة:
```bash
pip install -r requirements.txt
```

### س26: "FileNotFoundError: content.json"
**ج:** أنت في مجلد خاطئ:
```bash
cd ramadan_sports_bot
python ramadan_bot.py
```

### س27: "UnicodeEncodeError"
**ج:** مشكلة في ترميز الحروف العربية:
- تأكد من حفظ الملفات بـ UTF-8
- أضف في بداية السكريبت: `# -*- coding: utf-8 -*-`

### س28: البوت توقف عن العمل!
**ج:** تحقق من:
1. الإنترنت متصل؟
2. Terminal لم يُغلق؟
3. المفاتيح لم تنتهي صلاحيتها؟
4. راجع ملف الأخطاء (إن وُجد)

### س29: "Rate limit exceeded"
**ج:** تجاوزت حد Twitter:
- الانتظار 15 دقيقة
- التحقق من عدد الطلبات
- البوت يحترم الحدود تلقائياً

### س30: كيف أحدّث المشروع؟
**ج:**
1. حمّل النسخة الجديدة
2. احتفظ بملفك `.env`
3. عدّل `content.json` إذا لزم
4. أعد التثبيت

## 🔐 الأمان والخصوصية

### س31: هل مفاتيحي آمنة؟
**ج:** نعم، إذا:
- ✅ لم ترفع `.env` على GitHub
- ✅ لم تشارك المفاتيح مع أحد
- ✅ استخدمت `.gitignore` المرفق

### س32: ماذا لو تسربت مفاتيحي؟
**ج:** فوراً:
1. اذهب إلى Developer Portal
2. احذف المفاتيح القديمة (Revoke)
3. أنشئ مفاتيح جديدة
4. حدّث ملف `.env`

### س33: هل يمكن لأحد اختراق حسابي؟
**ج:** لا، إذا:
- حافظت على سرية المفاتيح
- استخدمت 2FA على حساب X
- لم تثبت برامج مشبوهة

### س34: هل البوت يقرأ رسائلي الخاصة؟
**ج:** لا! البوت يستخدم فقط صلاحية "Post Tweets" ولا يطلب أي صلاحيات أخرى.

### س35: هل بياناتي محفوظة؟
**ج:** نعم، كل شيء محلي:
- المفاتيح في `.env` على جهازك
- السجل في `posted_log.json`
- لا توجد خوادم خارجية

## 💡 نصائح متقدمة

### س36: كيف أشغّل البوت على سيرفر دائم؟
**ج:** استخدم systemd (Linux):
```bash
sudo cp systemd/ramadan-bot.service /etc/systemd/system/
sudo systemctl enable ramadan-bot
sudo systemctl start ramadan-bot
```

### س37: هل يمكن استخدام Docker؟
**ج:** نعم! أنشئ Dockerfile:
```dockerfile
FROM python:3.11-slim
WORKDIR /app
COPY . .
RUN pip install -r requirements.txt
CMD ["python", "ramadan_bot.py", "--auto"]
```

### س38: كيف أراقب البوت عن بُعد؟
**ج:** 
- استخدم `screen` أو `tmux`
- راقب logs: `tail -f bot.log`
- إشعارات Telegram (تحتاج تعديل)

### س39: هل يمكن جدولة منشورات مسبقة؟
**ج:** نعم، عدّل السكريبت لإضافة:
```python
schedule.every().day.at("09:00").do(post_day, 1)
schedule.every().day.at("21:00").do(post_day, 2)
```

### س40: كيف أضيف إحصائيات؟
**ج:** استخدم Twitter Analytics أو عدّل السكريبت لحفظ:
- عدد الإعجابات
- إعادات التغريد
- المشاهدات
